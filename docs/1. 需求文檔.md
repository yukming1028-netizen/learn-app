# 📄 需求文檔：親子學習互動平台（Parent-Child Learning Companion）

> **版本**：1.0  
> **作者**：Hermes Agent  
> **日期**：2026年5月22日  

---

## 1. 概述與愿景

本平台旨在打造一個**以家長為主導、子女為核心使用者**的智能學習互動系統，結合遊戲化任務機制與自適應題庫，幫助兒童建立持續學習習慣。  
通過「**無註冊綁定 + 家長管控 + 動態難度調整**」三大創新設計，降低使用門檻、強化親子互動、提升學習成效。

> ✅ **核心理念**：  
> *「孩子不需帳號，家長不必操心；任務自動適應，進步看得見。」*

---

## 2. 使用者角色

| 角色 | 描述 |
|------|------|
| **家長用戶（Parent）** | 負責註冊、綁定子女、設定學習計劃、查看數據報告、調整規則。可免費使用所有功能。 |
| **子女用戶（Child）** | 1~15歲兒童，無需註冊或記密碼，透過掃碼快速激活，僅與家長賬戶綁定後方可使用。 |

---

## 3. 核心功能需求（FR）

### ✅ 3.1 認證與綁定流程

| ID | 需求描述 | 優先級 |
|----|----------|--------|
| FR-01 | 家長端可**免費註冊帳號**（郵箱/手機 + 密碼），支援 Google/Facebook 登入 | P0 |
| FR-02 | 子女端**無需任何註冊流程**，首次啟動即進入「掃碼綁定模式」 | P0 |
| FR-03 | 家長端生成**一鍵綁定二維碼（QR Code）**，含唯一加密綁定令牌（UUID + 非對稱簽名） | P0 |
| FR-04 | 子女端透過相機掃描家長端 QR 碼後，自動建立綁定關係，並顯示「綁定成功」動畫與歡迎語 | P0 |
| FR-05 | 每個子女帳號僅能綁定**一個家長帳號**，但一個家長可綁定**最多 3 名子女** | P1 |
| FR-06 | 若子女端被重裝或換機，仍可用原 QR 碼重新掃碼恢復資料（雲端同步） | P1 |

> 🔐 **安全設計**：  
> - 所有綁定令牌使用 RSA 256 簽名，避免偽造  
> - 綁定過程無需輸入任何帳號密碼，僅靠加密令牌與設備唯一識別碼（UUID）

## 4. 家長端 Dashboard

| ID | 需求描述 | 優先級 |
|----|----------|--------|
| FR-07 | 提供**綜合數據看板**，顯示子女總學習時數、完成任務數、平均正確率 | P0 |
| FR-08 | 支援按**年級（小一至中三）**、**學科（數學、語文、英語、科學）**、**難度（簡單/中等/困難）** 進行多維度篩選與圖表分析 | P1 |
| FR-09 | 展示**每日 / 每週 / 每月**的學習趨勢曲線圖（如：使用時間 vs 正確率） | P1 |
| FR-10 | 提供「進步榜」功能，對比該子女過去 7 天/30 天的表現變化（文字 + 圖表） | P2 |
| FR-11 | 支援導出 PDF 報告（含圖表與評語），可列印或分享給家教、老師 | P2 |

## 5. 學習計劃管理（家長端）

| ID | 需求描述 | 優先級 |
|----|----------|--------|
| FR-12 | 家長可建立**每週計劃**與**每月計劃**，設定：每日建議學習時長、任務數量、推薦學科組合 | P0 |
| FR-13 | 支援「模板化計劃」：預設「小一數學強化」「英語每日10分鐘」等模板，家長可複製修改 | P1 |
| FR-14 | 計畫可設定**執行時段**（如週一至五 17:00–18:30），超出時間自動鎖定任務 | P1 |
| FR-15 | 家長可手動「暫停」或「延後」計劃，並接收子女未完成提醒（推送通知） | P2 |

## 6. 子女端學習體驗

| ID | 需求描述 | 優先級 |
|----|----------|--------|
| FR-16 | 子女端主介面顯示「今日任務」清單，自動載入家長設定的計劃任務 | P0 |
| FR-17 | 支援**額外自由探索**：子女可點擊「想玩別的」進入題庫商城，自由選擇其他學科/難度任務（無獎勵積分） | P1 |
| FR-18 | 每完成一題，即時給予**正向回饋**（動畫、音效、貼紙收集），累積可兌換虛擬道具（如帽子、寵物） | P1 |
| FR-19 | 題目難度根據以下指標**動態調整**：回答正確率、做題速度、學科弱項分析 | P0 |
| FR-20 | 錯誤題目自動加入「錯題本」，隔日或每週重複出現（間隔重複演算法） | P1 |

---

## 7. 動態難度與 AI 適應引擎

| ID | 需求描述 | 優先級 |
|----|----------|--------|
| FR-21 | 系統內建**自適應題庫引擎**，基於標籤化題目、學習者模型與預測演算法（如 IRT / Kalman Filter） | P0 |
| FR-22 | 每次作答後更新子女「知識掌握度」並預測下一題最適難度 | P0 |
| FR-23 | 支援**錯題重複機制**：錯誤題目隔 1 小時不重現，24 小時後低頻出現，連續 3 次正確移出錯題本 | P1 |

## 8. 非功能需求（NFR）

| ID | 描述 |
|----|------|
| NFR-01 | 跨平台支援：iOS / Android / Web（家長端）；Android / iPadOS（子女端優先） |

## 9. 技術架構建議

```
[子女端 App (Android/iOS)] ← HTTPS → [API Gateway]
                                      ↓
[家長端 Web/App] ←→ [Auth Service] ←→ [PostgreSQL DB]
                                      ↓
                             [Adaptive Engine (Python/ML)]
                                      ↓
                           [題庫服務（JSON + 标籤化）]
```

- **前端**：React Native 或 Flutter  
- **後端**：FastAPI / Node.js  
- **資料庫**：PostgreSQL  
- **雲端存儲**：AWS S3 或 Firebase Storage  
- **推送通知**：Firebase Cloud Messaging (FCM)  
- **AI 模型**：本地輕量模型 + 遠程訓練

## 10. 道德與教育設計原則

| 原則 | 實踐方式 |
|------|----------|
| ✅ **非商業化** | 無廣告、無內購、無付費關卡 |
| ✅ **防止成癮** | 每日最大時長限制（可設定）、自動休息提醒 |
| ✅ **正向激勵** | 只獎勵努力與進步，不獎勵「最快」或「最聰明」 |
| ✅ **尊重隱私** | 不記錄語音、不追蹤行為、不賣數據 |
| ✅ **親子共學** | 鼓勵家長偶爾參與子女任務（如一起解謎） |
