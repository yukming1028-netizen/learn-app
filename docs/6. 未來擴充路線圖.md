# 🌱 未來擴充需求與實施路線圖
## 親子學習互動平台 — 從 MVP 到教育科技標竿

> ✅ **目標**：從「單一應用」升級為「可持續、可擴展、有數據驅動力的兒童學習生態系統」

---

## 🚀 階段一：V2 - 增強核心體驗（3–6個月）

### 1. ✅ 家長端登入與認證系統

| 需求 | 說明 |
|------|------|
| 家長需使用郵箱/手機 + 密碼登入，取代 QR 綁定為唯一認證方式 | 目前僅靠綁定，無法管理多裝置或忘記密碼 |

#### 🛠️ 實施方案
- 增加 `/auth/login`（JWT）與 `/auth/register` API  
- 使用 `bcrypt` 加密密碼  
- 集成 Google/Facebook OAuth（未來擴充）  
- **整合點**：於 `parents` 表新增 `password_hash`, `email_verified`

#### 💡 價值
> 讓家長能跨裝置管理子女，提升使用黏著度與信任感。

#### ⏱️ 優先級：**P0**

---

### 2. ✅ 子女端「錯題本」自動複習系統

| 需求 | 說明 |
|------|------|
| 系統根據子女答錯題目，自動安排「間隔重複（Spaced Repetition）」復習任務 | 像 Anki 一樣智能提醒：第一次錯 → 1小時後再見 → 24小時 → 7天 |

#### 🛠️ 實施方案
- 新增表 `review_schedule`：
  ```sql
  CREATE TABLE review_schedule (
      child_id UUID REFERENCES children(id),
      question_id UUID REFERENCES questions(id),
      next_review_at TIMESTAMPTZ,
      interval_days INTEGER, -- 當前間隔天數
      ease_factor DECIMAL(3,2), -- 難度係數（1.3~2.5）
      PRIMARY KEY (child_id, question_id)
  );
  ```
- 每次答題後，根據 `is_correct` 與 `time_taken_sec` 更新算法  
- 子女端每日啟動時，自動載入「今日複習任務」清單

#### 💡 價值
> 科學提升記憶效率，讓學習從「練習」升級為「長期掌握」

#### ⏱️ 優先級：**P0**

---

### 3. ✅ 家長端「學習報告」PDF 匯出與分享

| 需求 | 說明 |
|------|------|
| 家長可點擊「下載月報」，生成含圖表、進步趨勢、弱項分析的 PDF 報告，支援 Email 或列印 |

#### 🛠️ 實施方案
- 後端新增 `/children/{id}/report.pdf` API  
- 使用 `ReportLab`（Python）或 `WeasyPrint` 生成 PDF  
- 圖表：用 `matplotlib` 或 `plotly` 動態繪製  
- 模板設計：簡潔、童趣風格，符合兒童教育美學

#### 💡 價值
> 讓家長能與老師、祖父母分享進步成果，建立正向反饋循環

#### ⏱️ 優先級：**P1**

---

## 🚀 階段二：V3 - 智慧化與社群化（6–12個月）

### 4. ✅ AI 生成題目引擎（AI-Powered Question Generator）

| 需求 | 說明 |
|------|------|
| 使用 LLM（如 Claude 3 / GPT-4o）根據課程綱要，自動產生**符合年級與難度的填空題、應用題** |

#### 🛠️ 實施方案
- 新增 `/generate-question` API  
- 輸入：`{"grade": "2", "subject": "math", "topic": "加法進位"}`  
- 輸出：標準 JSON 格式（符合 `questions` 表結構）  
- 人工審核機制：AI 生成題目需經「教育專家」標註後才上線

#### 💡 價值
> 解決題庫擴充瓶頸，讓系統能**自動適應新課程標準**（如十二年國教）

#### ⏱️ 優先級：**P1**

---

### 5. ✅ 教師上傳與校園模式

| 需求 | 說明 |
|------|------|
| 允許老師註冊帳號，上傳班級作業、測驗卷，並指定學生（子女）完成 |

#### 🛠️ 實施方案
- 新增 `teachers` 表 + `classrooms` 表  
- 教師可建立「作業任務」，綁定特定 `child_id`  
- 子女端顯示：「老師佈置的作業」專區  
- 家長端收到通知：「您的小孩有新作業待完成」

#### 💡 價值
> 從「家庭學習工具」升級為「家校協同平台」，具備學校採用潛力

#### ⏱️ 優先級：**P2**

---

### 6. ✅ 學習熱力圖與區域教育洞察（Data for Education Policy）

| 需求 | 說明 |
|------|------|
| 匿名彙總全平台數據，顯示「全國小學生最常出錯的數學知識點」 |

#### 🛠️ 實施方案
- 建立 `aggregated_insights` 表：每日聚合  
  ```sql
  CREATE TABLE aggregated_insights (
      date DATE,
      grade_level INTEGER,
      subject TEXT,
      topic TEXT,
      total_attempts BIGINT,
      wrong_rate DECIMAL(5,4)
  );
  ```
- 提供公開簡報頁面（如 `insights.learn-app.example.com`）  
- 與教育部/教研單位合作，提供教育政策建議

#### 💡 價值
> 從「個人學習工具」躍升為「公共教育資源」，具備社會價值與政府補助潛力

#### ⏱️ 優先級：**P2**

---

## 🚀 階段三：V4 - 生態系統化（1–3年）

### 7. ✅ 學習成就系統 + 虛擬寵物互動

| 需求 | 說明 |
|------|------|
| 子女完成任務 → 獲得「學習能量」→ 解鎖虛擬寵物、服裝、小遊戲 |

#### 🛠️ 實施方案
- 新增 `achievements` 表：`{child_id, badge_name, unlocked_at}`  
- 虛擬寵物為動畫 SVG，可點擊互動（如餵食、玩耍）  
- 所有獎勵**無實質價值**，純為正向激勵

#### 💡 價值
> 以遊戲化設計提升兒童主動性，同時避免「商業化」爭議

#### ⏱️ 優先級：**P3**

---

### 8. ✅ 跨平台同步與離線模式優化

| 需求 | 說明 |
|------|------|
| 子女端 App 支援完全離線作答，恢復網路後自動上傳紀錄 |

#### 🛠️ 實施方案
- 在 SQLite（本地）緩存 `child_progress` 記錄  
- 上傳時比對時間戳與伺服器版本，避免衝突  
- 使用 Firebase Offline Persistence 或 Redux Persist（Flutter）

#### 💡 價值
> 適用偏鄉、網路不穩地區，實現真正的教育平權

#### ⏱️ 優先級：**P1**

---

### 9. ✅ 多語言支援與文化適應（國際化）

| 需求 | 說明 |
|------|------|
| 支援英文、繁體中文、簡體中文、甚至泰語、越南語等東南亞語言 |

#### 🛠️ 實施方案
- 所有文字內容抽離為 `i18n/` JSON 檔案（如 `zh-TW.json`, `en-US.json`）  
- UI 根據裝置語言自動切換  
- 題目內容標註語言屬性：`language: 'zh-TW'`

#### 💡 價值
> 適合海外華人家庭、新住民子女，拓展市場與社會影響力

#### ⏱️ 優先級：**P3**

---

### 10. ✅ 家長社群 × 經驗分享平台（非商業化）

| 需求 | 說明 |
|------|------|
| 在 App 內建「家長話題區」，可分享學習心得、推薦資源、提問疑難 |

#### 🛠️ 實施方案
- 建立 `parent_forum` 表：`{parent_id, title, content, created_at}`  
- 所有內容**不開放廣告、不收費用、無點讚機制**，僅供交流  
- 由教育專家定期回覆高品質問題（如「孩子專注力差怎麼辦？」）

#### 💡 價值
> 構建**非商業、正向、互助的家長學習社群**，形成品牌忠誠度

#### ⏱️ 優先級：**P3**

---

## 📊 整體實施路線圖總覽（甘特圖概念）

| 階段 | 時間 | 核心目標 |
|------|------|----------|
| **V1（現有）** | 0–2月 | 基礎綁定 + 答題 + 題庫 |
| **V2** | 3–6月 | 登入系統、錯題本、PDF 報告 |
| **V3** | 7–12月 | AI 出題、教師模式、教育洞察 |
| **V4** | 1–3年 | 虛擬寵物、離線同步、多語言、社群 |

> ✅ 每個階段皆可獨立部署，不影響前一版本運作  
> ✅ 所有功能設計**嚴格避開商業化元素**（無廣告、無內購、無付費牆）

---

## 💬 最終理念：讓科技，成為親子間的橋樑

| 原則 | 實踐 |
|------|------|
| **非商業化** | 無廣告、無內購、不賣數據 |
| **正向激勵** | 只獎勵努力與進步，不比快、不比聰明 |
| **尊重隱私** | 不收集位置、不追蹤行為、符合 COPPA & GDPR |
| **教育本位** | 所有功能服務於「學習」本身，而非使用者停留時間 |
